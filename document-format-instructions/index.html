    4. 提示块说明 / Alerts / Callouts Reference

    This page defines the alert (callout) types used in this documentation.

    Alert definitions are provided bilingually (English / Chinese) for reference and consistency. In user-facing documents, alert content should remain in a single language.

    Note / 注意

    Note / 注意

    Supplementary information that may be useful.

    English\ Useful information that users should know, even when skimming content.

    中文\ 用于补充说明的信息，即使快速浏览文档，也建议读者留意。

    Tip / 提示

    Tip / 提示

    Optional advice that helps optimize usage.

    English\ Optional suggestions that help improve efficiency, performance, or user experience.

    中文\ 非必须内容，但有助于提升效率、性能或使用体验的建议。

    Important / 重要

    Important / 重要

    Key information required for correct operation.

    English\ Critical information that must be understood to ensure correct usage or configuration.

    中文\ 为确保正确使用或配置，必须理解和遵循的关键信息。

    Warning / 警告

    Warning / 警告

    Potential risks that could cause data loss or damage.

    English\ Indicates potential risks that may result in data loss, system damage, or safety issues.

    中文\ 提示可能导致数据丢失、系统损坏或安全问题的高风险情况。

    Caution / 谨慎

    Caution / 谨慎

    Situations that may lead to unexpected behavior.

    English\ Highlights conditions that may cause unexpected or undesired behavior. The risk level is lower than a Warning.

    中文\ 提示可能引发非预期行为的情况，风险等级低于“警告（Warning）”。

    Usage Guidelines / 使用规范

    • Use alerts only when they provide meaningful value to the reader.
    • Avoid overusing alerts; typically one or two per document section is sufficient.
    • Do not place alerts consecutively.
    • Do not nest alerts within other elements.
    • Use standard Markdown blockquotes for alerts to ensure cross-platform compatibility.
